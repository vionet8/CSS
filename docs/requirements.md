---

## 1. プロジェクト概要

### 目的

コンテンツ制作における「思考整理」「構造設計」「再利用」を支援する統合プラットフォーム。  
既存の動画編集ソフトやプレゼンソフトが苦手とする以下の領域をカバーする。

- 論理構造の整理
- 人気コンテンツの分析
- 認知変化の設計
- コンテンツの再利用

### 最重要コンセプト

> 本システムは「動画編集ソフト」でも「PowerPoint互換ソフト」でもない。  
> 目的は **「人の思考・認知構造を整理し、再利用可能なコンテンツへ変換すること」** である。

### 設計方針

**「編集」ではなく「構造化」を中心に設計する。**

## 2. 想定ユーザー

| タイプ | 主な用途 |
|--------|---------|
| YouTube投稿者 | 動画の構成設計・スライド下書き作成 |
| Note執筆者 | 記事の論理構造整理・再利用 |
| ブロガー | URL記事からの構造抽出・改稿 |
| SNS発信者 | 長文から要点抽出・スライド化 |
| 教育系コンテンツ制作者 | 認知変化を意識した教材設計 |

## 3. データ構造（最重要設計）

全コンテンツを以下の共通構造で保持する。これが唯一の正として管理される。

```yaml
project:
  id: uuid
  title: "プロジェクト名"
  description: "説明"
  raw_content: "入力テキスト（Markdown・テキスト等）"

  logic_structure:
    title: "コンテンツタイトル"
    thesis: "主な主張"
    nodes:
      - id: node_1
        type: 現状 | 問題 | 原因 | 解決策 | 根拠 | 具体例 | 結論
        title: "ノードタイトル"
        content: "詳細"
        children: [...]

  slides:
    - id: slide_1
      order: 1
      type: title | content | section | conclusion
      title: "スライドタイトル"
      body: "本文"
      speaker_notes: "発表者ノート"
      image_hint: "推奨画像の説明"

  assets:
    images: [...]
    videos: [...]
    audio: [...]
```

## 4. 機能要件

### Phase 1（実装済み）✅

#### 4.1 プロジェクト管理

| 機能 | 仕様 |
|------|------|
| 作成 | タイトル・説明を入力して新規作成 |
| 保存 | コンテンツ・構造・スライドを SQLite に永続化 |
| 読み込み | 一覧から選択して復元 |
| 削除 | プロジェクト単位で削除 |

#### 4.2 コンテンツ入力

| 入力形式 | 対応状況 |
|---------|---------|
| Markdown | ✅ テキストエリアに直接貼付け |
| テキスト | ✅ テキストエリアに直接貼付け |
| URL記事 | ✅ BeautifulSoup で本文抽出 |
| PDF | ✅ PyPDF2 でテキスト抽出 |
| Note記事コピー | ✅ テキストとして貼付け可能 |

#### 4.3 ロジック構造生成

- Claude API（claude-sonnet-4-6）で入力テキストを分析
- 抽出する要素: **主張・根拠・具体例・結論・現状・問題・原因・解決策**
- ツリー形式で表示（親子関係を色分けで視覚化）

**ノードタイプと色**

| タイプ | 色 |
|--------|---|
| 現状 | 青 |
| 問題 | 赤 |
| 原因 | オレンジ |
| 解決策 | 緑 |
| 根拠 | 紫 |
| 具体例 | 黄 |
| 結論 | ブランドカラー |

#### 4.4 スライド生成

- ロジック構造から Claude API でスライドを自動生成
- 生成内容: タイトル・サブタイトル・本文・発表者ノート・画像候補（テキストヒント）
- 最初にタイトルスライド、最後にまとめスライドを自動配置

#### 4.5 スライド編集

| 機能 | 仕様 |
|------|------|
| 手動編集 | タイトル・本文をインプレースで編集 |
| AI改善 | 自然言語指示でスライド単体を Claude が改善 |
| 順番変更 | （Phase 2 で DnD 実装予定） |

#### 4.6 画像管理

| 機能 | 仕様 |
|------|------|
| アップロード | ドラッグ＆ドロップ・複数同時対応 |
| リサイズ | 幅・高さ指定（Pillow LANCZOS） |
| トリミング | x/y/w/h 指定（Pillow） |
| 分割 | 2・3・4・6・9分割（グリッド計算） |

## 5. 非機能要件

| 項目 | 要件 |
|------|------|
| レスポンス | AI API 以外の操作は 500ms 以内 |
| AI レスポンス | タイムアウト 120秒（構造分析・スライド生成） |
| データ永続化 | SQLite ファイル（`css.db`）にローカル保存 |
| セキュリティ | API キーは `.env` で管理、Git 管理外 |
| ファイル管理 | アップロードファイルは `uploads/` に保存 |
